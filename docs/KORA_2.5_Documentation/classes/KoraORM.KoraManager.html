\KoraORM

\KoraORMKoraManager

The KoraManager class represents a single connection to a Kora backend.

In order to gain access to KoraEntities, one should use the methods found in this class. These methods will take steps to make sure that if you get two objects representing the same record in Kora, then they will use the same memory.

$metadataCache

$metadataCache : array

A cache for all the metadata obtained from loading custom KoraControls and KoraEntities, as well as the metadata generated automatically from the information stored within Kora.

This cache is accessed many times during the lifetime of this instance. It contains information about all the custom KoraControls and KoraEntities which were loaded. It also contains default metadata which is automatically generated by inspecting the Kora backend.

getByKid()

getByKid(mixed $kid, boolean $checkExists) : \KoraORM\a

Retrieve a KoraEntity with the specified KID.

If another KoraEntity with that KID was previously created, a reference to that KoraEntity is returned, otherwise, a new KoraEntity with the KID is created and then returned.

If $checkExists is true, then a Kora search is performed to ensure the object exists within Kora before create a new object.

Parameters

mixed $kid

the KID of the KoraEntity to retrieve
boolean $checkExists

whether to check if the KID exists in Kora first

Returns

\KoraORM\a —

KoraEntity instance on success, false on failure

parseKid(mixed $kid) : \KoraORM\a

Parse a KID in multiple forms to a standardized form.

createKoraEntity()

createKoraEntity(mixed $kid, boolean $checkExists, mixed $data) : \KoraORM\<code>false</code>

Create or retrieve an existing KoraEntity with the given KID.

If a KoraEntity was previously created with the given KID, a reference to the pre-existing KoraEntity is returned. Otherwise, a new KoraEntity is created.

If $checkExists is true, then a Kora Search is preformed to confirm that the corresponding record does in fact exist before creating a new KoraEntity.

If some data is specifiec, then the KoraEntity which is created with initially populated with that data. In this case, data should contain all of the fields the object may need during its lifetime.

Parameters

mixed $kid

the KID of the object to create
boolean $checkExists

whether to check if the KID exists in Kora first
mixed $data

the Kora data for the object, usually retrieved via a Kora search

Returns

\KoraORM\false —

on failure, a KoraEntity with the KID on success
